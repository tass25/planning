fig_macro_handling.png
Macro Integrity How Macros Are Kept Together
/* The parser tracks a macro stack macros are never split */ %MACRO outer(); ← stack: [outer] block OPEN %MACRO inner(); ← stack: [outer, inner] nested OPEN DATA work.x; SET src; RUN; ← RUN; does NOT close the macro %MEND inner; ← stack: [outer] inner CLOSED PROC PRINT; RUN; ← RUN; does NOT close the macro %MEND outer; ← stack: [] block CLOSED ✓ /* Everything between %MACRO outer and %MEND outer = ONE partition */
1. Balance Check count %MACRO vs %MEND before processing. Mismatch = reject immediately.
2. Macro Stack push on %MACRO, pop on %MEND. Block stays open while stack is non-empty.
3. Nesting Depth %DO/%IF increment, %END decrements. Inner constructs cannot break macro boundary.
4. Block-End Rules MACRO_DEFINITION only closes on %MEND. Never on RUN; or QUIT; from inner blocks.
5. Grammar Rule Lark LALR parser treats macro_body as a recursive container structurally atomic.
6. Cross-File Resolution macros defined in one file, called in another, are linked via the dependency graph.

Figure 4 Macro stack demo + six safeguards that guarantee macros are never split across partitions.

The agent controls retrieval: adaptive k, query reformulation, RAPTOR level escalation, retrieval retry
