### Task 2: BaseAgent Abstract Class

**File**: `partition/base_agent.py`

**What it must have**:
- Abstract `process()` method
- `agent_name: str` property
- `logger` bound with `structlog.get_logger().bind(agent=self.agent_name)`
- `trace_id: UUID` passed at construction or auto-generated
- `@with_retry` decorator (generic retry wrapper with exponential backoff)

**Key code pattern**:
```python
from abc import ABC, abstractmethod
import structlog
from uuid import uuid4, UUID

class BaseAgent(ABC):
    def __init__(self, trace_id: UUID | None = None):
        self.trace_id = trace_id or uuid4()
        self.logger = structlog.get_logger().bind(
            agent=self.agent_name,
            trace_id=str(self.trace_id)
        )

    @property
    @abstractmethod
    def agent_name(self) -> str: ...

    @abstractmethod
    async def process(self, *args, **kwargs): ...
```

**Retry decorator**:
```python
import asyncio
from functools import wraps

def with_retry(max_retries=3, base_delay=1.0, fallback=None):
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, **kwargs):
            for attempt in range(max_retries + 1):
                try:
                    return await func(*args, **kwargs)
                except Exception as e:
                    if attempt == max_retries:
                        if fallback:
                            return fallback(*args, **kwargs)
                        raise
                    delay = base_delay * (2 ** attempt)
                    await asyncio.sleep(delay)
        return wrapper
    return decorator
```

### Task 4: FileAnalysisAgent (#1)

**File**: `partition/entry/file_analysis_agent.py`

**Inputs**: `project_root: Path`  
**Outputs**: `list[FileMetadata]`

**Step-by-step logic**:

1. **Discover files**: `pathlib.Path(project_root).rglob("*.sas")` — collect all `.sas` files recursively.
2. **For each file**:
   - Read raw bytes: `file.read_bytes()`
   - Detect encoding: `chardet.detect(raw_bytes)['encoding']` — fallback to `'utf-8'` if `None`
   - Decode: `raw_bytes.decode(encoding, errors='replace')`
   - Compute SHA-256: `hashlib.sha256(raw_bytes).hexdigest()`
   - Count lines: `content.count('\n') + 1`
   - Lark pre-validation: run a minimal Lark grammar check for balanced `%MACRO/%MEND`, `DATA/RUN;`, `PROC/RUN;` pairs. Set `lark_valid=True/False` accordingly. Store error details in `lark_errors`.
   - Build `FileMetadata` object.
3. **Log**: For each file, log `file_path`, `encoding`, `line_count`, `lark_valid` via structlog.
4. **Return**: `list[FileMetadata]` — skip files with `lark_valid=False` (log as ERROR, but include in output for registry).

**Lark pre-validation grammar** (minimal — just checks balanced blocks):
```lark
start: block*
block: data_block | proc_block | macro_block | other_line
data_block: DATA_KW /[^;]*/ ";" _lines "RUN" ";"
proc_block: PROC_KW /[^;]*/ ";" _lines ("RUN" ";" | "QUIT" ";")
macro_block: MACRO_START /[^;]*/ ";" _lines MEND ";"
_lines: /.+/s
DATA_KW: /(?i)DATA\b/
PROC_KW: /(?i)PROC\b/
MACRO_START: /(?i)%MACRO\b/
MEND: /(?i)%MEND\b/
other_line: /[^\n]+/ NEWLINE
```

> Note: This is a **pre-validation only** — the full boundary grammar comes in Week 3–4. Here we just check that blocks are structurally closed.

**Tests** (`tests/test_file_analysis.py`):
- Test 1: Discover 3 `.sas` files in a temp directory → returns 3 `FileMetadata`
- Test 2: File with valid SAS → `lark_valid=True`
- Test 3: File with unclosed `%MACRO` → `lark_valid=False`, `lark_errors` not empty
- Test 4: File with Latin-1 encoding → correctly detected, `encoding='ISO-8859-1'`
- Test 5: SHA-256 dedup — same content → same `content_hash`

### Task 5: CrossFileDependencyResolver (#2)

**File**: `partition/entry/cross_file_dep_resolver.py`

**Inputs**: `files: list[FileMetadata]`, `project_root: Path`  
**Outputs**: Writes to `cross_file_deps` SQLite table

**Step-by-step logic**:

1. **Build file index**: `{normalised_path: file_id}` for all discovered files.
2. **For each file**, regex scan the decoded content for:
   - `%INCLUDE '(.+?)';` or `%INCLUDE "(.+?)"` → `ref_type='INCLUDE'`
   - `LIBNAME (\w+) '(.+?)';` → `ref_type='LIBNAME'`
   - `%INCLUDE &(\w+)` → `ref_type='INCLUDE'`, `resolved=False` (macro variable — unresolvable statically)
3. **Resolve paths**: For each match:
   - Try `(file_dir / ref_path).resolve()` — if it exists in the file index → `resolved=True`, `target_file_id=matched_id`
   - Try `(project_root / ref_path).resolve()` — same check
   - Else → `resolved=False`, `target_file_id=NULL`
4. **Write** each dependency row to `cross_file_deps` table via SQLAlchemy.
5. **Log summary**: `{total_deps_found}` total, `{resolved_count}` resolved, `{unresolved_count}` unresolved.

**Regex patterns**:
```python
import re

INCLUDE_PATTERN = re.compile(
    r"%INCLUDE\s+['\"]([^'\"]+)['\"]", re.IGNORECASE
)
LIBNAME_PATTERN = re.compile(
    r"LIBNAME\s+(\w+)\s+['\"]([^'\"]+)['\"]", re.IGNORECASE
)
MACRO_VAR_INCLUDE = re.compile(
    r"%INCLUDE\s+&(\w+)", re.IGNORECASE
)
```

**Tests** (`tests/test_cross_file_deps.py`):
- Test 1: File with `%INCLUDE 'utils/macros.sas';` where `utils/macros.sas` exists → `resolved=True`
- Test 2: File with `%INCLUDE &path;` → `resolved=False`
- Test 3: File with `LIBNAME mylib '/data/library';` → `ref_type='LIBNAME'` row created
- Test 4: No cross-file refs → zero rows inserted

**Inputs**: `files: list[FileMetadata]`  
**Outputs**: Rows in `file_registry` SQLite table

### Task 7: SQLite Database Setup

```python
from sqlalchemy import create_engine, event
from sqlalchemy.orm import sessionmaker, Session
from partition.entry.registry_writer_agent import Base  # imports all models

def get_engine(db_path: str = "file_registry.db"):
    engine = create_engine(f"sqlite:///{db_path}", echo=False)
    # Enable WAL mode for better concurrent read performance
    @event.listens_for(engine, "connect")
    def set_sqlite_pragma(dbapi_conn, connection_record):
        cursor = dbapi_conn.cursor()
        cursor.execute("PRAGMA journal_mode=WAL")
        cursor.execute("PRAGMA foreign_keys=ON")
        cursor.close()
    return engine

def init_db(engine):
    Base.metadata.create_all(engine)

def get_session(engine) -> Session:
    return sessionmaker(bind=engine)()
```

### Task 9: Data Lineage Tracking

**What**: Extend `CrossFileDependencyResolver` and the DB schema to track **data lineage** — not just file-to-file links, but dataset-to-dataset and column-level data flow.

**Why**: Your supervisor identified this as a key gap. File-level linking (`%INCLUDE` → file B) only answers "which files depend on which files?". Data lineage answers:
- **Table-level**: Which output datasets depend on which input datasets? (`DATA tgt.summary; SET staging.clean; MERGE staging.lookup;` → `summary` depends on `clean` + `lookup`)
- **Column-level**: Which output column was derived from which input columns? (`revenue = SUM(unit_price * quantity)` → `revenue` ← `unit_price`, `quantity`)
- **Cross-file data flow**: File A creates `work.temp`, File B reads `work.temp` → data lineage edge across files

**New DB table**: `data_lineage`

```python
class DataLineageRow(Base):
    __tablename__ = 'data_lineage'

    id = Column(Integer, primary_key=True, autoincrement=True)
    source_file_id = Column(String, ForeignKey('file_registry.file_id'), nullable=False)
    lineage_type = Column(String, nullable=False)      # TABLE_READ | TABLE_WRITE | COLUMN_DERIVE
    source_dataset = Column(String, nullable=True)      # e.g. 'staging.clean'
    target_dataset = Column(String, nullable=True)      # e.g. 'tgt.summary'
    source_columns = Column(Text, nullable=True)        # JSON list: ["unit_price", "quantity"]
    target_column = Column(String, nullable=True)       # e.g. 'revenue'
    transform_expr = Column(String, nullable=True)      # e.g. 'SUM(unit_price * quantity)'
    block_line_start = Column(Integer, nullable=True)   # line in source file
    block_line_end = Column(Integer, nullable=True)
```

**Regex patterns for table-level lineage**:
```python
# Detect dataset reads
SET_PATTERN = re.compile(r"\bSET\s+([\w.]+)", re.IGNORECASE)
MERGE_PATTERN = re.compile(r"\bMERGE\s+([\w.]+(?:\s*\([^)]*\))?(?:\s+[\w.]+(?:\s*\([^)]*\))?)*)", re.IGNORECASE)
FROM_PATTERN = re.compile(r"\bFROM\s+([\w.]+)", re.IGNORECASE)

# Detect dataset writes
DATA_OUTPUT_PATTERN = re.compile(r"\bDATA\s+([\w.]+(?:\s+[\w.]+)*)", re.IGNORECASE)
CREATE_TABLE_PATTERN = re.compile(r"CREATE\s+TABLE\s+([\w.]+)", re.IGNORECASE)
INSERT_INTO_PATTERN = re.compile(r"INSERT\s+INTO\s+([\w.]+)", re.IGNORECASE)
```

**Implementation approach**:
1. **Phase 1 (Week 1–2)**: Table-level lineage via regex — detect which datasets each block reads from and writes to. Store in `data_lineage` table.
2. **Phase 2 (Week 3–4)**: Column-level lineage — after Lark grammar is built, extract assignment expressions to track column derivations.

**Gold standard annotation extension** — `.gold.json` files should also include:
```json

  "data_lineage": [
    {
      "source_datasets": ["src.customers", "src.orders"],
      "target_datasets": ["work.active_customers"],
      "block_id": 2
    }
  ]
}
```

## Common Pitfalls

1. **Chardet confidence**: `chardet.detect()` sometimes returns `{'encoding': None, 'confidence': 0}` for small files. Always fallback to `utf-8`.
2. **Lark grammar**: Don't try to write the full SAS grammar now — just balanced block validation. The full grammar is Week 3–4.
3. **SQLite path**: Use absolute paths for `file_registry.db` to avoid issues when running tests from different directories.
4. **Gold standard quality**: Spend time on annotation. Every downstream metric (boundary accuracy, RAPTOR hit-rate) depends on this corpus. Budget 4+ hours.
5. **SHA-256 on bytes, not string**: Hash the raw bytes before decoding — this ensures encoding-independent dedup.
