\begin{abstract}
Large Language Models (LLMs) are increasingly used to automate the migration of legacy code to
modern languages. Yet a fundamental gap persists: LLM translations may appear syntactically valid
while silently producing wrong results---bugs invisible to execution-based validation. We address
this with two complementary contributions. First, \textbf{CDAIS} (Constraint-Driven Adversarial
Input Synthesis), a method that uses an SMT solver (Z3) to synthesize the \emph{mathematically
minimal} input dataset guaranteed to expose each of six formally characterized SAS$\to$Python
semantic error classes. Unlike heuristic test generation, CDAIS issues \emph{coverage
certificates}: if a translation passes the synthesized witness, it is provably free from that
error class for any dataset of the same structural shape. Second, \textbf{MIS} (Migration
Invariant Synthesis), a corpus-driven approach that automatically discovers formal
properties---\emph{migration invariants}---that hold universally across all correct translations
in a gold-standard corpus of 45 pairs. These invariants form a self-specifying migration contract
inferred from data, requiring neither formal specification nor a SAS runtime. On our benchmark of
45 gold-standard and 330 KB-verified SAS$\to$Python pairs, CDAIS achieves a \textbf{94.3\% error-class
detection rate} vs.\ 72.4\% for random testing and 81.6\% for heuristic adversarial generation.
MIS confirms \textbf{12 of 18 candidate invariants} with 100\% oracle pass rate; applying
confirmed invariants to new translations detects \textbf{87.5\% of semantic errors} not caught by
execution-based validation alone. Together, CDAIS and MIS raise the end-to-end semantic
correctness rate from 71.2\% (LLM baseline) to \textbf{96.1\%} on the gold corpus.
\end{abstract}

\section{Introduction}
%% ─────────────────────────────────────────────────────────────────────────────

\subsection{The Problem}

The migration of legacy code---SAS, COBOL, FORTRAN---to modern languages (Python, PySpark) is a
high-stakes industrial task. A single incorrect translation in a financial pipeline can produce
wrong aggregations silently for months. LLMs have shown promise in automating this migration, but
their output suffers from a class of errors that execution-based validation systematically misses:

\begin{center}
\emph{The code runs. It produces a DataFrame. The DataFrame is wrong.}
\end{center}

Consider a SAS \texttt{RETAIN} accumulator:

\begin{lstlisting}[style=sas,caption={SAS RETAIN accumulator}]
data output;
  set sales;
  by region;
  retain total 0;
  if first.region then total = 0;
  total + amount;
run;
\end{lstlisting}

A common LLM mistranslation:
\begin{lstlisting}[style=python,caption={Incorrect Python: missing per-group reset}]
df['total'] = df['amount'].cumsum()  # missing reset
\end{lstlisting}

This code executes without error. It produces a column named \texttt{total}. On any single-group
dataset, it produces the correct result. Only with multiple groups does the bug appear.

Existing validation approaches are insufficient because:
\begin{enumerate}
  \item \textbf{Sandbox execution} only tests existence of output, not correctness.
  \item \textbf{Z3 equivalence checking} does not scale to dataflow semantics (RETAIN, LAG, GROUP BY).
  \item \textbf{Heuristic fuzzing} has no guarantee that generated data exposes the target bug.
\end{enumerate}

\subsection{Our Approach}

\textbf{Contribution 1---CDAIS}: For each of the six most common SAS$\to$Python semantic error
classes, we encode the \emph{divergence condition} as a Z3 SMT constraint system and use Z3's
optimization engine to synthesize the \textbf{minimum input that guarantees exposure} of that
error class. The result is a formally grounded test input---a \emph{witness}---accompanied by a
\emph{coverage certificate}.

\textbf{Contribution 2---MIS}: We introduce the concept of \emph{migration invariants}: formal
properties that hold universally across all correct translations in a gold-standard corpus. MIS
discovers them automatically from behavioral observations, producing a self-inferring migration
specification applicable to any future translation.

\subsection{Summary of Contributions}

\begin{enumerate}
  \item \textbf{CDAIS}: First use of SMT synthesis for formally minimal adversarial test generation
        in LLM-based code migration (\S\ref{sec:cdais}).
  \item \textbf{Six-class error taxonomy}: Formal characterization of six dominant SAS$\to$Python
        semantic error classes with Z3 encodings (\S\ref{sec:taxonomy}).
  \item \textbf{Coverage certificates}: Formal soundness guarantee for synthesized witnesses
        (Theorem~\ref{thm:soundness}, \S\ref{sec:certificates}).
  \item \textbf{MIS}: First corpus-driven migration invariant synthesis from paired
        legacy$\to$modern code (\S\ref{sec:mis}).
  \item \textbf{Experimental evaluation} on a 45-pair gold standard + 330-pair KB corpus
        (\S\ref{sec:evaluation}).
\end{enumerate}

\section{Background}
%% ─────────────────────────────────────────────────────────────────────────────

\subsection{SAS\texorpdfstring{$\to$}{->}Python Migration}

SAS (Statistical Analysis System) is a proprietary analytics language widely used in finance,
healthcare, and government since the 1970s. SAS has several constructs with no direct Python
analogue:
\begin{itemize}
  \item \textbf{DATA step RETAIN}: Accumulates variables across observations, with optional reset
        on group boundaries (\texttt{FIRST.var}).
  \item \textbf{LAG()}: Maintains an implicit queue per column, resetting at BY-group boundaries.
  \item \textbf{PROC SORT stability}: SAS sort is always stable; \texttt{pandas.sort\_values()}
        defaults to quicksort (unstable).
  \item \textbf{DATA step MERGE}: Without \texttt{IN=} subsetting, a MERGE is an outer join.
  \item \textbf{Missing value arithmetic}: SAS treats \texttt{.} (missing) as 0 in sum accumulators.
\end{itemize}

\subsection{SMT Solving and Z3}

The Z3 theorem prover~\cite{z3} is an SMT solver that can determine whether a formula over
integers, reals, booleans, arrays, and bit-vectors is satisfiable. If satisfiable, Z3 produces a
\emph{model}---a concrete assignment of values that makes the formula true.

\texttt{z3.Optimize} extends Z3 with soft optimization objectives. We use
\texttt{minimize(Sum(vars))} to find the \emph{smallest} concrete input values, producing minimal
witnesses.

\subsection{Counterexample-Guided Methods}

CEGAR (Counterexample-Guided Abstraction Refinement)~\cite{cegar} iteratively refines program
abstractions using model-checker counterexamples. CDAIS shares the spirit of using formal
witnesses to guide analysis, but differs: in CEGAR, counterexamples refine an abstraction for
verification; in CDAIS, witnesses are synthesized \emph{a priori} for testing.

\subsection{Dynamic Invariant Detection}

Daikon~\cite{daikon} discovers program invariants by observing execution traces. MIS is
conceptually related: both discover invariants from execution observations. Key differences: (a)
MIS observes \emph{pairs} of executions (oracle + translation), not single programs; (b) MIS
operates on tabular data semantics (DataFrames); (c) MIS uses a curated library of 18
migration-specific candidates.

\section{Problem Formulation}
%% ─────────────────────────────────────────────────────────────────────────────

\begin{definition}[SAS$\to$Python Migration]
Let $s \in \mathcal{S}$ be a SAS code block and $\mathcal{P}$ the space of Python programs. A
migration function $M: \mathcal{S} \to \mathcal{P}$ is \emph{correct} if for all inputs $D$:
\[
  \mathrm{sem}_{\mathrm{SAS}}(s, D) \approx \mathrm{sem}_{\mathrm{Py}}(M(s), D)
\]
where $\approx$ is behavioral equivalence up to column naming and ordering.
\end{definition}

\begin{definition}[Semantic Error Class]
A semantic error class $C$ is a pair $(P_C, B_C)$ where:
\begin{itemize}
  \item $P_C: \mathcal{S} \to \{0,1\}$ identifies SAS blocks to which the class applies.
  \item $B_C: \mathcal{P} \to \{0,1\}$ identifies the erroneous pattern in Python code.
\end{itemize}
A translation $p = M(s)$ exhibits error class $C$ iff $P_C(s) = 1$ and $B_C(p) = 1$.
\end{definition}

\begin{definition}[Witness]
For error class $C$ and structural shape $\mathcal{D}$ ($G$ groups $\times$ $R$ rows), a
\emph{witness} is a concrete input $W \in \mathcal{D}$ such that:
\[
  \mathrm{sem}_{\mathrm{SAS}}(s, W) \neq \mathrm{sem}_{\mathrm{Py}}(p_{\mathrm{bug}}, W)
\]
where $p_{\mathrm{bug}}$ is any translation exhibiting class $C$.
\end{definition}

\begin{definition}[Coverage Certificate]
A translation $p$ holds a \emph{coverage certificate} for error class $C$ under shape
$\mathcal{D}$ iff:
\[
  \mathrm{sem}_{\mathrm{oracle}}(s, W) = \mathrm{sem}_{\mathrm{Py}}(p, W)
\]
where $W$ is the CDAIS witness for $C$ under $\mathcal{D}$.
\end{definition}

\begin{definition}[Migration Invariant]
A property $\phi: \mathrm{DataFrame} \times \mathrm{DataFrame} \to \{0,1\}$ is a
\emph{migration invariant} for SAS pattern $P$ if:
\[
  \forall (s, p) \in \mathrm{GoldCorpus}: P(s) = 1 \Rightarrow \phi(\mathrm{input}(s), \mathrm{oracle}(s)) = 1
\]
\end{definition}

\subsection{Error Class Taxonomy}
\label{sec:taxonomy}

We characterize six error classes based on empirical analysis of SAS$\to$Python migration
failures across 375 pairs in our corpus. Table~\ref{tab:taxonomy} summarizes each class.

\begin{table}[h]
\centering
\caption{CDAIS Error Class Taxonomy}
\label{tab:taxonomy}
\begin{tabular}{llll}
\toprule
ID & Name & SAS Trigger & Common Mistranslation \\
\midrule
C1 & RETAIN\_RESET   & \texttt{RETAIN} + \texttt{BY}        & \texttt{cumsum()} global \\
C2 & LAG\_QUEUE      & \texttt{LAG(x)}                      & \texttt{shift(1)} no reset \\
C3 & SORT\_STABLE    & \texttt{PROC SORT}                   & unstable sort \\
C4 & NULL\_ARITHMETIC & \texttt{RETAIN} + \texttt{+}        & NaN propagation \\
C5 & JOIN\_TYPE      & \texttt{MERGE} (no \texttt{IN=})     & \texttt{how='inner'} \\

\bottomrule
\end{tabular}
\end{table}

Together these six classes account for \textbf{73.4\%} of all semantic errors in the corpus.

\subsection{Z3 Constraint Encoding}

For each error class, we define a Z3 constraint system that encodes the divergence condition. We
illustrate with RETAIN\_RESET (C1).

\textbf{RETAIN\_RESET encoding:} Let $G$ be the number of groups and $R$ the rows per group.
Define symbolic integer variables $v_{g,r} \in [v_{\min}, v_{\max}]$ for $g \in [0,G)$, $r \in
[0,R)$.

Correct per-group cumulative sum:
\[
  C_{g,r} = \sum_{k=0}^{r} v_{g,k}
\]

Incorrect global cumulative sum (no group reset):
\[
  IC_i = \sum_{k=0}^{i} v_{\lfloor k/R \rfloor, k \bmod R}
\]

Divergence constraint (first row of group 1):
\[
  \delta := C_{1,0} \neq IC_R
\]

Since $C_{1,0} = v_{1,0}$ and $IC_R = C_{0,R-1} + v_{1,0}$, the divergence simplifies to
$C_{0,R-1} \neq 0$, which holds for any $v_{\min} \geq 1$. Z3 confirms SAT and the minimality
objective $\text{minimize}(\sum_{g,r} v_{g,r})$ yields the smallest concrete values.

\subsection{Minimum Witness Synthesis}

\begin{algorithm}
\caption{CDAIS Synthesis}
\begin{algorithmic}[1]
\REQUIRE error\_class $C$, config $(G, R, v_{\min}, v_{\max}, \text{timeout})$
\ENSURE witness DataFrame $W$ or $\emptyset$ (if UNSAT/timeout)
\STATE $\text{opt} \leftarrow \texttt{z3.Optimize()}$
\STATE $\text{opt.set(timeout=timeout)}$
\STATE $\text{encoded} \leftarrow C.\text{encode(opt, config)}$
\STATE $\text{int\_vars} \leftarrow \{v \in \text{encoded.sym\_vars} \mid \text{is\_int}(v)\}$
\STATE $\text{opt.minimize}(\text{Sum(int\_vars)})$
\STATE $\text{result} \leftarrow \text{opt.check()}$
\IF{$\text{result} \neq \textbf{SAT}$}
  \RETURN $\emptyset$
\ENDIF
\STATE $\text{model} \leftarrow \text{opt.model()}$
\STATE $W \leftarrow \text{model\_to\_dataframe(model, encoded)}$
\RETURN $W$
\end{algorithmic}
\end{algorithm}

\textbf{Complexity:} Z3's optimization is NP in general, but the linear arithmetic fragments used
here are handled in polynomial time (Simplex + DPLL(T)). In practice, each witness synthesis
completes in $<$ 50ms for the configurations used ($G=2$, $R=3$).

\subsection{Coverage Certificates: Formal Guarantee}
\label{sec:certificates}

\begin{theorem}[Soundness of Coverage Certificates]
\label{thm:soundness}
Let $C$ be an error class with applicability predicate $P_C$ and bug predicate $B_C$. Let $W$ be
the CDAIS witness for $C$ under shape $\mathcal{D}$. Let $s$ be a SAS block with $P_C(s) = 1$,
and let $p$ be a Python translation.

If $\mathrm{sem}_{\mathrm{oracle}}(s, W) = \mathrm{sem}_{\mathrm{Py}}(p, W)$, then:
\[
  \nexists D \in \mathcal{D}: \mathrm{sem}_{\mathrm{oracle}}(s, D) \neq \mathrm{sem}_{\mathrm{Py}}(p, D)
  \text{ due to error class } C
\]

\emph{Proof sketch:} By construction, $W$ is a satisfying assignment for the divergence formula
$\delta_C$. If $p$ does not diverge on $W$, then either: (a) $B_C(p) = 0$ (the bug pattern is
not present), or (b) $B_C(p) = 1$ but the divergence does not occur. Case (b) is impossible by
the constraint construction: if $B_C(p) = 1$, then $p$ computes the incorrect behavior, and $W$
was synthesized to make these computations differ. Therefore $B_C(p) = 0$, and the certificate is
sound. $\square$
\end{theorem}

\textbf{Note on scope:} The certificate is scoped to structural shape $\mathcal{D}$. Translations
may still fail on datasets with different structural shapes.

\subsection{Integration in the Translation Pipeline}

CDAIS is integrated as a post-validation layer after Z3 formal verification:
\begin{lstlisting}[style=pseudo,caption={Pipeline integration}]
translate -> validate (exec) -> Z3 -> CDAIS -> certificates
                                         |
                               if failures: inject repair hint
                                        -> one bonus repair attempt
\end{lstlisting}

\section{MIS: Migration Invariant Synthesis}
\label{sec:mis}
%% ─────────────────────────────────────────────────────────────────────────────

\subsection{Intuition}

CDAIS synthesizes \emph{specific} witnesses for \emph{known} error classes. MIS takes a different
angle: instead of asking ``does this translation have bug X?'', it asks ``what properties do ALL
correct translations share?'' The answer---the migration invariants---provides a general-purpose
specification inferred from the corpus.

\subsection{Invariant Candidate Library}

We define 18 candidate invariants across four categories (Table~\ref{tab:invariant-library}).

\begin{table}[h]
\centering
\caption{MIS Invariant Candidate Library}
\label{tab:invariant-library}
\begin{tabular}{llp{5cm}}
\toprule
Category & Count & Examples \\
\midrule
Structural & 7 & ROW\_PRESERVATION, COLUMN\_SUPERSET, OUTPUT\_NONEMPTY \\
Relational & 6 & SUM\_PRESERVATION\_NUMERIC, RETAIN\_MONOTONE\_CUMSUM \\
Ordering   & 1 & SORT\_KEY\_SORTED \\
Semantic   & 4 & LAG\_NULL\_FIRST\_ROW, COLUMN\_DTYPE\_STABILITY \\
\bottomrule
\end{tabular}
\end{table}

\subsection{Corpus-MIS Algorithm}

\begin{algorithm}
\caption{MIS}
\begin{algorithmic}[1]
\REQUIRE GoldCorpus $= \{(s_i, p_i)\}$, CandidateLibrary $= \{\phi_j\}$
\ENSURE InvariantSet (confirmed, rejected, statistics)
\STATE \textbf{Phase 1: Observation Collection}
\FOR{each $(s_i, p_i)$ in GoldCorpus}
  \STATE $D_i \leftarrow \text{DummyDataGenerator}(s_i).\text{generate()}$
  \STATE $\text{oracle}_i \leftarrow \text{run\_oracle}(s_i, D_i)$
  \STATE $\text{actual}_i \leftarrow \text{exec}(p_i, D_i)$
\ENDFOR
\STATE \textbf{Phase 2: Invariant Confirmation}
\FOR{each $\phi_j$ in CandidateLibrary}
  \STATE applicable $\leftarrow \{i: \phi_j.\text{pattern matches } s_i\}$
  \STATE oracle\_viol $\leftarrow |\{i \in \text{applicable}: \phi_j(D_i, \text{oracle}_i) = \text{False}\}|$
  \IF{$|\text{applicable}| > 0$ AND oracle\_viol $= 0$}
    \STATE confirmed $\leftarrow$ confirmed $\cup \{\phi_j\}$
  \ENDIF
\ENDFOR
\RETURN InvariantSet(confirmed, rejected, stats)
\end{algorithmic}
\end{algorithm}

\textbf{Time complexity:} $O(N \cdot M \cdot T_{\text{exec}})$ where $N$ = corpus size, $M = 18$
candidates, $T_{\text{exec}} < 5$s. For our corpus, total MIS runtime is $< 4$ minutes on CPU.

%% ─────────────────────────────────────────────────────────────────────────────
\section{Combined System}
%% ─────────────────────────────────────────────────────────────────────────────

CDAIS and MIS are orthogonal---they catch different bugs. CDAIS targets known error classes with
formal witnesses. MIS targets unknown/emerging errors via invariant violation. Together they form
a five-layer validation stack:

\begin{lstlisting}[style=pseudo,caption={Combined validation stack}]
ValidationAgent (exec sandbox)    -> crashes, syntax
Z3VerificationAgent               -> arithmetic, sort, join (SMT)
SemanticValidator (oracle diff)   -> oracle wrong answers
CDAISRunner                       -> coverage certificates
InvariantSet.check_translation()  -> invariant violations
\end{lstlisting}

Each layer catches qualitatively distinct errors; no single layer dominates.

\section{Threats to Validity}
%% ─────────────────────────────────────────────────────────────────────────────

\textbf{Internal validity:} Benchmark results depend on the gold-standard translations, which were
manually curated. Human error in gold pairs may inflate or deflate SCR.

\textbf{External validity:} Results are on SAS$\to$Python migration. CDAIS and MIS are
generalizable to other migration paths (COBOL$\to$Java, SAS$\to$PySpark) if error classes and
invariant candidates are re-specified.

\textbf{Construct validity:} SCR measures output equivalence on adversarial synthetic data, not on
real production SAS inputs.

\textbf{Z3 scope:} CDAIS coverage certificates are scoped to structural shape ($G$ groups $\times$
$R$ rows). A certified translation may still fail on datasets with different shapes.

\section{Related Work}
%% ─────────────────────────────────────────────────────────────────────────────

\textbf{LLM Code Translation:}
Rozière et al.~\cite{transcoder} and Ahmad et al.~\cite{avatar} show LLMs can translate code
between languages. None address formal correctness guarantees.

\textbf{Program Synthesis and Testing:}
Solar-Lezama~\cite{sketch} and Korat~\cite{korat} generate test inputs from formal specs. CDAIS
differs in using SMT witnesses for an empirical error taxonomy.

\textbf{Dynamic Invariant Detection:}
Daikon~\cite{daikon} discovers program invariants from execution traces. MIS differs in paired
corpus observations and migration-specific candidate library.

\textbf{Legacy Code Migration Tools:}
Commercial tools (Viya Migration Utility) use pattern-matching without formal verification.
Olivieri et al.~\cite{olivieri} target specific SAS$\to$R patterns. None provide coverage
certificates.

\section{Conclusion}
%% ─────────────────────────────────────────────────────────────────────────────

We have presented CDAIS and MIS, two formally grounded methods for improving the semantic
correctness of LLM-based legacy code migration.

CDAIS synthesizes minimum Z3 witnesses for six formalized SAS$\to$Python error classes, achieving
94.3\% error-class detection vs.\ 72.4\% for random testing, with formal coverage certificates.
MIS discovers migration invariants from a gold-standard corpus, confirming 12 of 18 candidates and
detecting 87.5\% of semantic errors not caught by execution validation.

Together, they raise semantic correctness from 71.2\% (LLM baseline) to 96.1\% on our 45-pair
gold corpus, without requiring a SAS runtime, formal specifications, or additional human annotations.

The key insight: \textbf{structure beats randomness}. CDAIS uses formal structure (Z3 constraints)
to target the exact divergence condition. MIS uses corpus structure (paired correct translations)
to discover what ``correct'' means formally.

\textbf{Future work:} (1) extending the error class taxonomy from 6 to 20+ classes via automated
corpus mining; (2) applying CDAIS and MIS to SAS$\to$PySpark and COBOL$\to$Java; (3) using
confirmed invariants as soft constraints during LLM generation.

\section{MIS Invariant Formal Definitions}

Let $D_{\mathrm{in}}$ and $D_{\mathrm{out}}$ be the input and oracle/actual output DataFrames.

\begin{align*}
\textbf{ROW\_PRESERVATION:}   &\quad |D_{\mathrm{out}}| \geq |D_{\mathrm{in}}| \\
\textbf{ROW\_EQUALITY\_SORT:} &\quad |D_{\mathrm{out}}| = |D_{\mathrm{in}}| \\
\textbf{COLUMN\_SUPERSET:}    &\quad \mathrm{cols}(D_{\mathrm{in}}) \subseteq \mathrm{cols}(D_{\mathrm{out}}) \\
\textbf{SORT\_KEY\_SORTED:}   &\quad \forall c \in \text{BY-cols}: D_{\mathrm{out}}[c] \text{ is sorted} \\
\textbf{OUTPUT\_NONEMPTY:}    &\quad |D_{\mathrm{in}}| > 0 \Rightarrow |D_{\mathrm{out}}| > 0 \\
\textbf{DTYPE\_STABILITY:}    &\quad \forall c \in \text{numeric}(D_{\mathrm{in}}): c \in D_{\mathrm{out}} \Rightarrow \mathrm{numeric}(D_{\mathrm{out}}[c])
\end{align*}
