Multi-Tier LLM Chain — Fallback & Verification

Primary → fallback chain. Each tier is only invoked if the previous fails (timeout, 429, circuit-open).

🟡 Tier 3 — Fallback 2 + Judge Groq
llama-3.3-70b-versatile
Circuit breaker: 3 failures → open 120s · 3-key rotation (300K TPD) · Also: cross-verify + accuracy judge
FAIL → all providers exhausted
🔴 PARTIAL status No LLM
Block marked PARTIAL · human review flag set · pipeline continues with remaining blocks
Quality Gates
1. Cross-Verification
Second independent LLM call with cross_verify.j2 — compares original SAS semantics vs translated Python, returns confidence 0–1
threshold: 0.7
↓ if confidence < 0.7
2. Reflexion Retry
reflection.j2 — shows the LLM its own output + cross-verify critique, asks it to correct specific issues. Max 2 retries.
↓ always
3. ValidationAgent Sandbox
ast.parse() + exec() in isolated subprocess (multiprocessing.Process, 15s timeout Windows). Auto-namespace provides synthetic DataFrames for any unknown SAS dataset names.
↓ after all blocks merged
4. Accuracy Judge (NEW)
Groq LLaMA-3.3-70b called independently (not the translator) with scoring rubric:
• Semantic equivalence: 50 pts
• Completeness: 25 pts
• Correctness: 25 pts
Returns JSON {"score": 82, "reason": "..."}
